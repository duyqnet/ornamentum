Server-side Basic Data Binding

Server side data binding allow users to bind a larger data stream which is available in the server side.

Unlike the client side data binding, whenever user performs a keyword search or user scrolls down to see more items, a new api request will be initiated to fetch data from the server which matches to the particular action.

What You Will See

Basic Usage

Server side data binding is handled via onDataBind callback property which is available within the dropdown features. This property allows users to customize the way of data binding inside a callback function. This call back function accepts DropdownRequestParams parameter (optional) and returns any generic type(T) of item set as an Observable(Data stream) DropdownQueryResult<T>.

When the data set is huge, without getting whole lot of data at once to the client side, data will be fetched chunk by chunk when user scrolled down to the last item.

Whenever user scrolls down to the last item to see more data, a new request will be sent to server. Therefore, it is mandatory to set true for loadOnScroll property when server side data binding is used.

In the above example, open Network tab in the chrome developer tools and search for item or scroll down, to see the initiated request as described earlier.
