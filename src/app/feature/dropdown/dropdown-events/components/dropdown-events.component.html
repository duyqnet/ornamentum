Dropdown Events

  • There are several events types are available in this dropdown as follows.
      • Triggers when initializing the dropdown.
      • Triggers when dropdown item select state changes. To identify selected items uniquely it is necessary to specify [selectTrackBy]" property. Therefore, a unique value key name should be set as the value for this attribute. According to this example it is 'key'. There are set of configuration are introduced in the dropdown to work smoothly with the selected change event. Those configurations can be seen as follows.
        • [triggerSelectChangeOnInit]="true or false" => The default value for this config is false.
        • [triggerSelectChangeOnFirstOptionSelect]="true or false" => The default value for this config is false.
        • [triggerSelectChangeOnModelUpdate]="true or false" => The default value for this config is false.
      • This will trigger after completion of data binding.
  • Please refer following example to see, how this event can be used and what types of data is passed to these events.
