DropDown Translations

  • Dropdown translations will allow users to configure the basic settings.
  • Those translations are required to passed via the translations attribute as follows.
    [translations]="dropdownTranslations"
  • Going through the following, shows the configurations can be passed via the translations.
    • noDataMessage
    • filterPlaceholder
    • selectedItemWrapPlaceholder
    • selectPlaceholder
  • Let's see how these can be configured in different scenarios.

Dropdown Translations for No Data Scenario

  • In this scenario, when data is not available users can provide customized message needs to show, via this translations property.
  • noDataMessage: "Customized No Data Message" => This configuration is used to set the message header name on the message to be shown in the dropdown body when there are no data to display. The default value for this config is 'No Results Available'

Dropdown Additional Translations

  • The other translations properties can be considered in this section.
    • filterPlaceholder: "This is filter place holder" => This configuration is used to customize the place holder text for dropdown value filtering text box when filtering is enabled. The default value for this config is 'Search'.
    • selectPlaceholder: "This is select place holder" => This configuration is used to customize the select place holder text in the dropdown. The default value for this config is 'Select'.
    • selectedItemWrapPlaceholder: "Wrapped Items" => This works only when multi selection mode enabled in the dropdown. For this purpose in here, selection mode is set for Multi. Also to enable selected items wrapping, it is required to set true for wrapDisplaySelectLimit. Refer Dropdown Item Selection's Multi Item Selection with Wrapping Selected Options section for more details. This configuration is used to customize the text which shows to indicate the wrapped items. The default value for this config is 'Item'. Please select multiple options from the drop down to see how this actually works.
