Server-side Data Binding

Server side data binding allow users to bind a data set that are not available in client side as a static data set but available in external data source with dynamic features. Now, server side data binding technique is available to handle and process data in this kind of scenarios.

Server side data binding allow users to bind a larger data stream which is available in the server side.

Unlike in Client-side Data Binding, whenever user performs a filtering, sorting, data pagination action a new API request will be initiated to fetch data from the server which matches to the particular action.

Basic Usage

Server side data binding is handled via onDataBind callback property which is available within the data table features. This property allows users to customize the way of data binding inside a callback function. This call back function accepts DataTableRequestParams parameter (optional) and returns any generic type (T) of item set as an Observable (Data stream) DataTableQueryResult<T>.
