Cell Template

Ornamentum allows flexible customization of the table cells of the Grid (excluding the header and footer section) or to format the data the table cells display.

If your going to use cell templating for more than one column you have to handel rospan on that cell manually, Please refer to Product Type & Revenue column in What You Will See example.

What You Will See

Basic Usage

Following code snippet depicts the integration of cell template. To define the cell template, nest an <ng-template> tag with the ngDataTableCell directive inside a <ng-data-table-column> tag. The template context is set to the current data item and the following additional fields are passed:
let-column="column" let-row="row" let-spanIndex="spanIndex" let-rowSpan="rowSpan"

Useful Properties

There are few more configurations available which affects the cell templating behaviours.

Column

column: DataTableColumnComponent Target Component: <ng-template> inside <ng-data-table-column>

The current column instance which contains all column properties. Use it as an alias for a template variable by utilizing the let-column="column" syntax.

Row

row: DataTableRow Target Component: <ng-template> inside <ng-data-table-column>

The current row instance which contains all row data. Use it as an alias for a template variable by utilizing the let-row="row" syntax.

Usage of row property

Span Index

spanIndex: number Target Component: <ng-template> inside <ng-data-table-column>

Span index of the current row which is useful when using row grouping. Use it as an alias for a template variable by utilizing the let-spanIndex="spanIndex" syntax.

Row Span

rowSpan: number Target Component: <ng-template> inside <ng-data-table-column>

Row Span of the current row which is useful when using row grouping. Use it as an alias for a template variable by utilizing the let-rowSpan="rowSpan" syntax.
