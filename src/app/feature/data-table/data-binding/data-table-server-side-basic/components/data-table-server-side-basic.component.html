Server-side Basic Data Binding

Server side data binding allow users to bind large data sets that are not directly accessible as a full collection in client side due to various reasons. Server side data binding can improve performance drastically for large data sets but integration is fairly complex compared to client side direct data binding.

Server side data binding allow users to bind a larger data stream which is available in the server side.

Unlike in Client-side Basic Data Binding, server side rendering will request for data on table initialize and whenever user performs filtering, sorting, page change, etc. operations. Integration party has to write the logic to fetch data via a backend service depending on the request context.

Basic Usage

Server side data binding is handled via onDataBind callback input property. This property allows users to customize the data binding logic within the callback function. This callback function will be triggered on table initialize and on dynamic data table operations. This function will receive DataTableRequestParams argument and expect to return RxJS Observable query result DataTableQueryResult<T> object stream. Data returned from this callback function will be directly bound to table assuming that it matches the on demand request context.
