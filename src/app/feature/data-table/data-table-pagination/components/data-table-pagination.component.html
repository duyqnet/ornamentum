Pagination

Most of the time it turns out when amount of data to be displayed didn't fit on a single page, or the page becomes ridiculously long. In such cases, you might want to turn on pagination.

The Pagination controls will be displayed on the bottom of the data table. you can set the number of rows to be displayed on the grid page and navigation controls.

What You Will See

Basic Usage

Depends on offset and limit values Pagination can be enable by binding a true into pageable property.

In default, pageable property is configured to false therefore, data table doesn't paginate itâ€™s contents, even pageable depended properties are defined. Those properties will be discuss on next section.

Useful properties

There are few more properties which effect to behaviour of pagination controls and those are applicable in global level of data table configuration after configuring the pageable property.

Page

page: number  
Default Value: 1

The page property allow to control which page the table starts off showing by calculate offset depending on provided page number, from the declared page users can paginate into different data views with in the declared page Limit. make sure to not set offset explicitly when page is in used.

Limit

limit: number  
Default Value: 10

Controls how many rows are shown per page and the rest of the data row's will be paginate into next page(s). If limit is not provided by the configuration the default value will be applied.
