Data Table Pagination

Most of the time it turns out that the amount of data to be displayed can’t fit on a single page, or the page becomes ridiculously long. In such cases, you might want to turn pagination on.

The Pagination controls will be displayed on the bottom of the grid. user can set the number of rows displayed on the grid page and navigation controls.

What You Will See


Basic Usage

To enable pagination, simply set pageable to true and by default pageable is false.

{{demoSnippet.pagination}}

In default the table doesn't paginate it’s contents, even if pageable depended properties are set. those properties will be discuss on Useful Properties section.

Useful properties

There are few more properties which effect the behaviour of pagination controls and those are applicable in global level of grid configuration after enabling the pageable.

Page

[page]: number = 'page number to view'
Default Value: 1

The page property lets you to control which page the table starts off showing, from the declared page users can paginate into different data views with in the declared data limit.

Limit

[limit]: number = 'number of data rows'
Default Value: 10

Controls how many rows are shown per page and the rest of the data row's will be paginate into series of grid pages. if limit is not provided by the configuration the default limit will be applied.
