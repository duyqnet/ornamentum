Data Loading Spinner

Data loading spinner is responsible to display a spinner on data table body while on data fetch operation, once after the success data retrieval process it will disappear and data will be displayed on the table body.

Loading spinner is very useful when on Server Side Basic Data Binding and Server Side Web Socket Data Binding operations. by default loading spinner is enabled and ornamentum will keep indicate the end user that the data retrieval process is still going on and improving user experience.

Basic Usage

By default loading spinner is enabled, Loading spinner can me disabled by set false value to showLoadingSpinner property in table level in the data table configuration.

showLoadingSpinner: boolean   Default Value: true Target Component: <ng-data-table>
