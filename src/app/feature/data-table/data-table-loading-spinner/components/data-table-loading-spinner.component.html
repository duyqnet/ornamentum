Data Table Loading Spinner

Loading Spinner is responsible to display a spinner on data table body while on data fetch operation, once after the success data retrieval process it will disappear and data will be displayed on the table body.

Loading spinner is very useful when on Server Side Data Binding and Real Time Data Binding operations. from enabling data loading spinner ornamentum will keep updates the end user that the data retrieval process is still going on and preventing affect to user experience negatively.

What You Will See

Basic Usage

Loading spinner can be enable by binding true to showLoadingSpinner property in global level of data table configuration (ng-data-table).

{{demoSnippet.loadingSpinner}}

Please note to display the loading spinner it is mandatory to enabling showLoadingSpinner property and after enabling, you can bind a custom template into the data loading scenario if you want. for loading spinner templates please refer Loading Spinner Template section.

Loading spinner property can be override in module level. for more details about Module level property overriding please refer Module Level Property Override section.

Useful Properties

As explain in the above, there is a property that affect to the behaviour of table loading spinner.

Show Loading Spinner

showLoadingSpinner: boolean = true or false
Default Value: false

Show Loading Spinner is responsible to continuously display default spinner on data table body until the data fetch operation is complete.
