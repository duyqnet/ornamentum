Column Sorting

This feature enables column data sorting according to the selected sorting order in specified column fields. Following configuration types are available when working with data table column sorting.

There are 03 types of sorting orders, which are:

  1. Ascending : column data will be sort according to ascending order.
  2. Descending : column data will be sort according to descending order.
  3. None (Default Sorting Order) : column data will be displayed as it is.

What You Will See

Basic Usage

Column sorting can be enabled in any data table column by declaring sortable to true otherwise it will be false by default.

By default the table does not sort itâ€™s contents. Also there are set of additional properties can be uses with sortable property and those will be explain in next section.

After enabling the sortable users can simply change the sorting order to any above mention 3 types, by clicking on icon that appears on data table column top right side corner.

Useful properties

There are few more properties that affects to the behaviour of column sorting. All the available properties can be applied to each table column by independently excepts Multi Column Sortable property.

Sort Order

sortOrder: string = 'asc' | 'desc' | ''  
Default Value: ''

This property allow users to order the data that already fetched or fetching from a data source. If sorting order is not mentioned by the user, then the default sorting mechanism will be applied for those sortable enabled columns.

Sort Field

sortField: string  
Default Value: 'same column field name'

Sort Field can be override by providing a different column field name, by default sort field value is same to it's column name. providing a different column name to the sort field, ornamentum will start sorting according to the provided column's data.

Multi Column Sortable

multiColumnSortable: boolean  
Default Value: false

Multi column sortable can be only use inside the data table global level configuration. If multi column sortable is enabled, data sorting will be process based on previously preserved column sorting order.
