Data Table Translations

  • Data table translations will allow users to configure the basic settings.
  • Those translations are required to passed via the translations attribute as follows.
    [translations]="DataTableTranslations"
  • Going through the following, shows the configurations can be passed via the translations.
    • noDataMessageHeader
    • noDataMessageBody
    • paginationLimit
    • paginationRange
  • Let's see how these can be configured in different scenarios.

Data Table Translations for No Data Scenario

The default template will be shown when data is not available. In that scenario, users can customized the message needs to show using this translations property.

  • noDataMessageBody: "Customized No Data Message Body" => This configuration is used to set the message header name on the message to be shown in the table body when there are no data to display. The default value for this config is 'No data to display. Added data will appear here.'
  • noDataMessageHeader: "Customized No Data Message Header" => This configuration is used to set the message to be shown in the table body when there are no data to display.The default value for this config is 'Whoops!'.

Data Table Translations for Pagination

Also this translations property can be used to customize the pagination naming. The following properties shows what are those.

  • paginationLimit: "This is the message for paginationLimit" => This is used to customize the pagination limit naming. The default value for this config is 'Limit'.
  • paginationRange: "This is the message for paginationRange" => This is used to customize the pagination range naming. The default value for this config is 'Results'.
