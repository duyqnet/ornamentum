Pagination

Pagination feature allows you to bind render large data sets without a performance impact while reducing the unnecessarily long scrollbars. This feature can be used with all types of data binding technique described in data binding section. The pagination controls are displayed in the table footer section.

What You Will See

Basic Usage

Pagination is disabled by default; hence, it can be enable by setting the [pageable] input property true in <data-table> component. Pagination functionality depends on [page] and [limit] input property values described later in this section.

Useful Properties

There are other useful input properties exposed in <data-table> component which can be used to change the default behaviour of the pagination functionality. These property values are applicable only when pagination is enabled.

Page

page: number   Default Value: 1

Page input property can be used to declare default the page to be rendered on init.

Limit

limit: number   Default Value: 10

Limit input property can be used to declare how many rows should be displayed per page.

Data Binding Support

Pagination functionality works with all types of data binding techniques described in data binding section.

  • Client Side Basic Data Binding & Client Side Stream Data Binding - Pagination will be handled within the data table by default.
  • Server Side Basic Data Binding & Server Side Web Socket Data Binding - Pagination associated parameters will be passed to the [onDataBind] callback input property where they can be used to fetch the target page associated data accordingly. Refer relevant data binding documentation for more information.
