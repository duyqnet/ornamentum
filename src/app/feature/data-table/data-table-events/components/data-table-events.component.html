Events List

All the available data table events can be detailed as follows. Please interact with the above example to see the outputs of the below mentioned events.

    • Triggers when initializing the data table.

      • {{dataTableInitData}}

    • This will trigger when row select change happen for all rows.
    • Click on the select all check box to see the output.

      • {{allRowSelectChangeEventData}}

    • Also this will emit all the selected row ids as an array if the rowSelectChange event is bind.
    • For each table row this will trigger when binding data for the available rows. That is the only occasion this event will be fired. Refer following output,

      • {{id}}

    • This event will be called after the columnBind, init & dataBound events.
    • Triggers when row select state changes. To identify selected rows uniquely it is necessary to specify [selectTrackBy]" property. Therefore, a unique value column name should be set as the value for this attribute. According to this example it is 'id'.
    • Click on the row select check box to see the output.

      • // click on the any row select box(es) to see the output {{rowSelectChangeEventData|json}}

    • If multi row selection option is enabled, the output is shown as an array of ids. Else it will be a single value(id).
    • For each table row cell this will trigger when binding cell data.
    • That means after a each rowBind event, this event will be called again & again for all the available cells in a given row.
    • That's why following output shows the same string(ex: cell bind event is called for row id => 10) 8 (available cells in a particular row in the following example) times per row.

      • {{item}}

    • Triggers when clicks on table cell.
    • Click on the cell to see the output.

      • {{cellClickEventData}}

    • Triggers for each column before all the other events to set column configurations.

      • {{item}}

    • As mentioned earlier, this is the first which called before all the other events.
    • This will trigger after completion of data binding.

      • {{dataBoundEventData}}

    • This event will be called after the columnBind and init events.
    • Triggers when clicking table column header row.

      • {{headerClickEventData}}

    • Triggers when user clicks on table row.

      • {{rowClickData}}

    • Triggers when user double clicks on table row.

      • {{rowDoubleClickData}}
